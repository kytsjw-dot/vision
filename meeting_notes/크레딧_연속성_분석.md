# 크레딧 연속성 분석

## 사용자 질문 정리
- **상황**: 같은 프로젝트를 회사와 집(현재)에서 동시에 개발
- **회사**: 퇴근 전까지 97K 크레딧 사용
- **집(현재)**: 56.9K 사용됨
- **의문**: 두 환경의 크레딧이 연속성이 있는지?

---

## 결론: 연속성 없음 (별개 계정)

### Kilo Code 크레딧 특성
| 항목 | 설명 |
|------|------|
| **계정 단위** | VS Code 확장 프로그램별 개별 계정 |
| **연동 여부** | 회사 PC ↔ 집 PC 연동되지 않음 |
| **크레딧 공유** | 불가능 (별도 과금) |
| **대화 기록** | 로컬 저장 (Git으로 동기화 필요) |

### 증거
- 회사: 97K 사용
- 집(현재): 56.9K 사용
- **숫자가 다름** = 별개 계정

---

## 문제점

### 1. 비용 중복 발생
- 회사에서도 Kilo Code 사용료 지불
- 집에서도 Kilo Code 사용료 지불
- **동일한 개발 보조 도구를 두 번 비용 지불**

### 2. 대화 맥락 단절
- 회사에서 나눈 대화 내용이 집에 없음
- 집에서 새로 시작하니 맥락을 다시 설명해야 함
- **프로젝트 컨텍스트 불일치**

---

## 해결 방안

### 방안 1: Git 동기화 (권장)
**회사 ↔ 집 동기화 방법:**
```
회사 PC
  ↓ (commit & push)
GitHub/GitLab 저장소
  ↓ (pull)
집 PC
```

**동기화 항목:**
- [ ] `meeting_notes/` 폴터 전체
- [ ] 코드 소스
- [ ] 설정 파일

**단계:**
1. 회사/집 모두 Git 설치
2. 원격 저장소 생성 (GitHub/GitLab)
3. 작업할 때마다 commit → push
4. 다른 환경에서 pull 받아 작업

### 방안 2: 한 환경에서만 개발
- 회사에서만 Kilo Code 사용
- 집에서는 코드만 작성 (AI 대화 없음)
- **비용 절감** but **생산성 저하**

### 방안 3: API 키 공유 (위험)
- 같은 Kilo Code 계정 사용
- **보안상 비권장**
- 회사 정책 위반 가능성

## 권장 워크플로우

### 회사에서 작업 시
```bash
# 작업 완료 후
git add .
git commit -m "회사: 기능 분석 완료"
git push origin main
```

### 집에서 작업 시
```bash
# 작업 시작 전
git pull origin main

# 작업 완료 후
git add .
git commit -m "집: 개발 환경 설정"
git push origin main
```
