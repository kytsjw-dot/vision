## 개발 전략

### 1단계: 회사 컴퓨터로 테스트 (1주)
**목표**: 로컬 개발 가능 여부 확인

**테스트 항목**:
- [ ] 로컬 이미지 생성 (Stable Diffusion)
- [ ] 로컬 TTS (Coqui TTS 등)
- [ ] 영상 렌더링 속도 측정
- [ ] 저장공간 사용량 확인

**결정 기준**:
- ✅ 속도/품질 만족 → 로컬 개발 계속
- ❌ 속도/품질 불만족 → 클라우드 API 사용

### 2단계: 클라우드 API 연동 (2주)
**목표**: Vertex AI Studio 모델 연동

**우선순위**:
1. **Gemini** - 대본 생성 (핵심 기능)
2. **Imagen 4** - 이미지 생성
3. **Veo 3** - 영상 생성 (고급 기능)
4. **Claude** - 고품질 대본 (옵션)

### 3단계: 하이브리드 아키텍처 (3주)
**목표**: 로컬 + 클라우드 병행

**구조**:
```
[사용자] → [우리 툴] → [로컬 처리] 또는 [클우드 API]
                    ↓
            [로컬]: 기본 기능, 빠른 처리
            [클우드]: 고급 기능, 고품질 결과
```

## 지원금 사용 우선순위

### 1순위: Google 지원금 (40만원) - 빠른 사용
**사용처**:
- OpenAI API (GPT-4, DALL-E)
- ElevenLabs (TTS)
- 기타 외부 API

**이유**: 선착순/한정 지원금, 먼저 소진 필요

### 2순위: Vertex AI Studio (140만원) - 올해 내 사용
**사용처**:
- Gemini API
- Imagen 4 API
- Veo 3 API
- Claude API (Vertex 통해)

**이유**: 12월 만료, Google 생태계 활용

## 비용 효율화 전략

### 로컬 vs 클라우드 선택 기준

| 상황 | 로컬 사용 | 클라우드 사용 |
|------|-----------|---------------|
| **개발/테스트** | ✅ 묶음 | ❌ 비용 발생 |
| **빠른 결과 필요** | ✅ 즉시 처리 | ⚠️ 네트워크 지연 |
| **고품질 결과 필요** | ❌ 품질 제한 | ✅ 최신 모델 |
| **대량 처리** | ✅ 비용 묶음 | ❌ 비용 증가 |
| **특수 기능** | ❌ 미지원 | ✅ 최신 기능 |

### 캐싱 전략
- 생성된 이미지/영상 캐싱
- 동일 프롬프트 재사용
- API 호출 최소화
