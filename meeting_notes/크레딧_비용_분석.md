# 크레딧 및 비용 분석

## 현재 상황

### 크레딧 비교
| 환경 | 크레딧 | 비고 |
|------|--------|------|
| 회사 | 90 | T-Flow 유료 플랜 |
| 현재 (Kilo Code) | $0.15 (54.6K 토큰) | AI 대화 비용 |

**중요**: 회사의 "90 크레딧"과 Kilo Code의 "$0.15"는 **완전히 다른 개념**입니다.

---

## 크레딧 종류 설명

### 1. T-Flow 크레딧 (회사의 90 크레딧)
- **의미**: T-Flow 플랫폼의 유료 사용권
- **용도**: 
  - AI 대본 생성
  - TTS 음성 생성
  - AI 이미지 생성
  - 영상 렌더링
- **특징**: T-Flow 서비스 내에서만 사용 가능
- **비용**: 월 구독제 또는 크레딧 충전제

### 2. Kilo Code 토큰 (현재 $0.15)
- **의미**: AI 어시스턴트(Kilo Code) 사용 비용
- **용도**: 
  - 코드 작성
  - 문서 작성
  - 대화/상담
- **특징**: 개발 보조 도구 사용료
- **비용**: 사용한 토큰량에 따라 과금

---

## 개발에 미치는 영향

### 직접적인 영향: 없음
Kilo Code의 $0.15는 **개발 보조 도구 비용**일 뿐, 개발하는 툴(T-Flow 유사 툴)의 기능과는 무관합니다.

### 간접적인 영향: 있음
개발할 툴에서도 **AI API 비용**이 발생합니다:

| API 서비스 | 용도 | 예상 비용 |
|------------|------|-----------|
| OpenAI GPT-4 | 대본 생성 | $0.03~0.06/1K 토큰 |
| OpenAI DALL-E 3 | 이미지 생성 | $0.04~0.08/이미지 |
| ElevenLabs | TTS 음성 | $0.30/1만 문자 |
| Azure Speech | TTS 음성 | $1/1시간 |

---

## 개발 방향 제안

### 옵션 1: API 키 연동 방식 (권장)
- 사용자가 자신의 API 키를 입력
- 개발자는 플랫폼만 제공, API 비용은 사용자 부담
- **장점**: 운영 비용 없음, 무제한 확장 가능
- **단점**: 사용자 진입장벽 약간 높음

### 옵션 2: 크레딧 시스템 구축
- 개발자가 API 키를 보유
- 사용자는 크레딧을 구매하여 사용
- **장점**: 수익 모델 확보, 사용자 편의성
- **단점**: API 비용 선투자 필요, 관리 복잡

### 옵션 3: 하이브리드
- 기본 기능은 묶음 제공
- 고급 기능은 API 키 연동 또는 크레딧 구매
- **장점**: 유연한 비즈니스 모델

---

## 결론

**회사 크레딧(90)과 현재 Kilo Code 비용($0.15)은 관련이 없습니다.**

개발할 툴의 비용 구조는 **별도로 설계**해야 합니다:
1. 사용자가 직접 API 키 입력 (개발자 부담 최소)
2. 크레딧 시스템 구축 (수익 모델)
3. 하이브리드 방식 (권장)
